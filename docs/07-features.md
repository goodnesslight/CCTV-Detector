# 07. Функціональні можливості

Інтерфейс організовано як `QTabWidget` з шістьма вкладками.

## 7.1. Вкладка «Пряма трансляція» (Live)

**Файл:** `app/ui/tabs/live_tab.py`

### 7.1.1. Контроли

```
┌─────────────────────────────────────────────────────────────┐
│ Джерело: [USB-камера ▼] [0_______]                          │
│ ☑ Люди (YOLO)  ☑ Обличчя (YuNet+SFace)  ☐ Трекінг + loitering│
│ [ Підключитися ]                                             │
├─────────────────────────────────────────────────────────────┤
│ Підключено: USB camera 0 | детектори: YOLO + YuNet+SFace    │
│ людей: 2 | знайомих: 1 | у зоні: 1                           │
├─────────────────────────────────────────────────────────────┤
│ ⚠ Тривале перебування в зоні: Каса (5.2с, трек #7)          │
├─────────────────────────────────────────────────────────────┤
│ [────── Відеокадр з overlay'єм ──────]                       │
│                                                              │
│      ┌──────────┐                                            │
│      │ Каса     │                                            │
│      │ ┌──────┐ │                                            │
│      │ │      │ │   ┌──────────┐                             │
│      │ │ #7   │ │   │ Іван 0.51│                             │
│      │ │ Іван │ │   │ ┌──────┐ │                             │
│      │ │      │ │   │ │ face │ │                             │
│      │ │      │ │   │ └──────┘ │                             │
│      │ └──────┘ │   └──────────┘                             │
│      └──────────┘                                            │
└─────────────────────────────────────────────────────────────┘
```

### 7.1.2. Особливості

- **Lazy завантаження моделей.** Кнопка «Підключитися» завантажує лише ті
  моделі, чиї чекбокси активні.
- **Persist чекбоксів.** Стан зберігається в `Settings.live_*_enabled` між
  запусками.
- **Status bar з підрахунками.** Лічильник людей, знайомих/чужих обличь, у скількох зонах хтось є.
- **Banner алертів.** Червона смуга з текстом події, автоматично ховається через 5 сек.
- **Camera flash.** Червона рамка навколо віджету camera_view протягом 0.3 сек при алерті.

### 7.1.3. Підтримувані джерела

- **USB-камера** (`int` index у `cv2.VideoCapture`).
- **RTSP-потік** (URL формату `rtsp://user:pass@host:port/stream`).

## 7.2. Вкладка «Персони»

**Файл:** `app/ui/tabs/persons_tab.py`

```
┌────────────────────────────────────────────────────────────┐
│ [Завантажити базу облич]                                   │
│ [Додати персону...] [Додати фото до вибраної] [Видалити]    │
├────────────────────────────────────────────────────────────┤
│ Персон: 3 | фото: 7                                        │
├────────────────────────────────────────────────────────────┤
│ Іван      —  3 фото  | появи: 12  | останній раз: 2 хв тому│
│ Олена     —  2 фото  | появи: 5   | останній раз: 35 хв тому│
│ Юрій      —  2 фото  | появи: 0   | останній раз: ніколи   │
└────────────────────────────────────────────────────────────┘
```

### 7.2.1. CRUD whitelist'у

- **Додати персону:** ім'я + одне фото (jpg/png/...). Обличчя витягається,
  embedding зберігається.
- **Додати фото:** для збільшення робастності — ще одне фото тієї ж людини
  (різний ракурс / освітлення).
- **Видалити:** разом з усіма фото та статистикою появ.

### 7.2.2. Лічильник появ

- `появи: N` — скільки разів цю людину впізнали (з 30-секундним дебаунсом).
- `останній раз: ...` — формат залежить від давності:
  - `< 1 хв` → «щойно»
  - `< 1 год` → «X хв тому»
  - `< 24 год` → «X год тому»
  - інакше → `DD.MM.YYYY HH:MM`

### 7.2.3. Live-оновлення

При відкритті вкладки + далі кожні 3 сек, поки вкладка видима. Перемикання
на іншу вкладку зупиняє таймер (економія CPU).

### 7.2.4. Файлова організація

```

├── Іван/
│   ├── photo1.jpg
│   ├── photo2.jpg
│   └── photo3.jpg
├── Олена/
│   ├── work.jpg
│   └── party.png
└── Юрій/
    ├── selfie.jpg
```

### 7.3.1. Робота користувача

1. Натиснути «Зробити знімок з камери» — береться поточний кадр з Live.
2. Створити нову зону, дати ім'я.
3. ЛКМ-кліки на канві — точки полігона.
4. ПКМ — закрити полігон.
5. Backspace — видалити останню точку.

### 7.3.2. Збереження

Авто-збереження на додавання/видалення/редагування. Файл `data/zones.json`:

```json

  { "name": "Серверна", "points": [[450, 100], [600, 100], [600, 250], [450, 250]] }
]
```

## 7.4. Вкладка «Події»

**Файл:** `app/ui/tabs/events_tab.py`

Журнал усіх алертів. Поєднує функції журналу, перегляду кліпів, галереї
знімків невпізнаних облич та пошуку по фото.

```
┌──────────────────────────────────────────────────────────────────┐
│ Тип: [Усі ▼] [Оновити] [Пошук по обличчю...] [Очистити пошук]    │
│                                          [Очистити журнал]        │
├──────────────────────────────────┬───────────────────────────────┤
│ Час        | Тип       | Опис   │ ┌─────────────────────────┐   │
│────────────|───────────|────────│ │                         │   │
│ 14:23:45   | unknown_..│ в кадрі│ │   [snapshot обличчя]    │   │
│ 14:18:12   | zone:Каса │ Каса   │ │   200×200 px            │   │
│ 14:15:00   | loiter... │ Каса.. │ │                         │   │
│            |           │        │ └─────────────────────────┘   │
│            |           │        │ ─── Кліп-плеєр ──────────     │
│            |           │        │ ▶ ━━━━━━━ 00:03 / 00:07      │
│            |           │        │                                │
│            |           │        │ [Додати до персон]            │
│            |           │        │ Тип: unknown_face              │
│            |           │        │ Час: 14:23:45                  │
└──────────────────────────────────┴───────────────────────────────┘
```

### 7.4.1. Фільтри

- **Тип події**: всі різні `kind` беруться з БД динамічно.
- **Період**: задається у статистиці; журнал показує останні 500 записів.

### 7.4.2. Знімок обличчя

Коли вибрано подію `unknown_face`, праворуч відображається JPG з обличчям
людини (з 20% padding для контексту: волосся, плечі). Знімок зберігається
в `data/unknown_faces/` під час спрацювання алерту, а шлях — у колонці
`events.snapshot_path`.

Якщо знімок відсутній (інші типи подій) — показується placeholder
"(знімок обличчя відсутній)".

### 7.4.3. Перегляд кліпу

Під знімком — `QMediaPlayer` віджет. При виборі події з кліпом — кліп
автоматично завантажується і починає програватись.

### 7.4.4. Кнопка «Додати до персон»

Активна тільки коли в обраної події є snapshot. Клік → запит імені →
система копіює знімок у whitelist (`data/known_faces/<name>/`) та
перебудовує SFace embedding'и. Наступна поява тієї самої людини буде
розпізнана як `known_face`.

### 7.4.5. Пошук по обличчю

Кнопка **«Пошук по обличчю...»** → файловий діалог → завантажити фото.
Система:

1. Прогоняє YuNet+SFace на завантаженому фото → отримує query-embedding.
2. Робить `find_by_embedding(query, threshold=0.4)` — векторизований
   cosine-пошук серед усіх `face_embedding` у БД.
3. Замінює таблицю на результати, додає шосту колонку **«Схожість»** з
   відсотком матчу (від 100% до 40%).
4. Кнопка **«Очистити пошук»** повертає звичайний журнал.

Корисний кейс: «знайди коли цей незнайомець був тут раніше» → завантажити
скрин із сьогоднішнього алерту → знайти всі попередні появи.

### 7.4.6. Експорт PDF

Викликає `app/reports/pdf_exporter.py`. Звіт включає:
- Метадані (період, версія, час генерації).
- Зведення: усього подій, по типах, по зонах.
- Журнал (до 1000 останніх записів).

### 7.6.1. Privacy blur

Якщо ввімкнено — у Live кадрах та у збережених кліпах bbox'и `unknown_face`
розмиваються через pixelate (12×12). Незнайомі обличчя стають нерозпізнаваними,
але людина видна як силует — алерт спрацьовує, журнал заповнюється, але
персональні дані сторонніх захищені (GDPR-сумісний режим).

Застосовується після перепідключення джерела (читається у момент створення
`DetectionPipeline`).

### 7.6.2. Теплова карта

Якщо ввімкнено — у Live кадри додається кольоровий overlay JET-палітри:
червоне = найбільш «протоптана» зона, синє = ніхто не ходить. Сітка живе
у `data/activity_heatmap.npy`, накопичується між запусками.

**«Скинути теплову карту»** обнуляє накопичену карту (наприклад при зміні
розміщення меблів, коли стара карта вже не відповідає).

При натисканні «Зберегти» — запис у `data/settings.json` + `Services.apply_settings()`,
що пропагує зміни в живі компоненти (детектори, AlertManager, ClipManager,
SightingsTracker).

## 7.8. Закриття застосунку

При закритті:

```python
def closeEvent(self, event):
    for tab in tabs:
        tab.cleanup()       # стопимо QThread, звільняємо камеру
    services.shutdown()      # закриваємо ClipManager, TTS, ...
    super().closeEvent(event)
```

Кожна вкладка має `cleanup()` (опційно), наприклад `LiveTab` зупиняє
`VideoWorker` і звільняє `cv2.VideoCapture`.
